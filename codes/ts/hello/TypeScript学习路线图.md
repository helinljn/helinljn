# TypeScript 学习路线图

## 第一阶段：先建立边界

先读 `01_基础篇` 和 `02_JS运行时语义篇`。目标是理解：TS 不负责运行时内存管理、网络、文件或线程模型；这些来自 JS 语义和 Node/Web 平台。

C++ 背景重点关注：

- `number` 不区分整数和浮点。
- 对象默认按引用共享，不是结构体值复制。
- `const` 约束绑定，不代表深度不可变。
- TS 类型检查会被擦除，运行时没有接口对象。

## 第二阶段：掌握类型系统日常用法

学习 `03_类型系统篇` 到 `05_面向对象篇`。目标是能写出清晰的 API 边界。

重点：

- 联合类型和类型收窄。
- `interface` 与 `type` 的使用取舍。
- 泛型约束与 C++ 模板的差异。
- 结构化类型与类继承的差异。

## 第三阶段：进入工程实践

学习 `06_模块与工程篇` 到 `10_TS6工程约束篇`。目标是能看懂现代 Node/TS 工程。

重点：

- ESM 的 `import/export`。
- NodeNext 模块解析。
- `.d.ts` 只是类型声明，不是实现。
- Node API 与 Web API 的边界。
- TypeScript 6.0 是本教程的编译器版本边界。

## 第四阶段：做练习和项目

先运行练习：

```bat
npm test
```

再阅读 `11_项目实战`：

- 文件批处理 CLI：目录遍历、文本转换、参数解析。
- 日志分析 CLI：解析、聚合、报告输出。

这两个项目刻意保持零运行时第三方依赖，重点放在 TS 类型、Node API 和可测试代码组织。
